Constraints on the use of the Service Access from WebSphere MQ: one-way pattern

A number of constraints apply when you use this pattern.

You can use the Service Access from WebSphere MQ: one-way pattern only if you meet all of the following constraints:

  • Requests are received as WebSphere MQ XML messages.
  • Requesting applications can generate an XML message that can be used as the body of a request to the target service.
  • The target service requests and responses use CCSID 1208.
  • WSDLs must contain the definitions that define the properties for the SOAPRequest node in the definitions of the top level WSDL. Therefore, some complex nested WSDLs cannot automatically be processed by the pattern.

    If WSDLs that do not meet this constraint are used, the pattern does not configure the SOAPRequest node, but lists configuration of the SOAPRequest node as a task to complete after generation of the pattern instance.

  • The pattern defines its own format for the logging and error messages; it is assumed that these messages are sufficient to meet the requirements in your own environment.

Some of the preceding constraints, particularly the CCSID, can be overcome by the use of a transformation. The Request Processor subflow is included as the place where any required transformation is added.
